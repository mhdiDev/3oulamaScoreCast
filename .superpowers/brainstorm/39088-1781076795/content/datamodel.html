Schéma de base de données

PostgreSQL · 8 tables principales

USER
  • id, email, username, passwordHash
  • avatarUrl, locale (fr/en)
  • role: USER | ADMIN
  • emailNotifications: boolean
  • pushSubscription: json
  • createdAt
INVITATION
  • id, token (unique, UUID)
  • createdBy → User
  • usedBy → User (nullable)
  • expiresAt, usedAt
TEAM
  • id, name, code (ex. FRA)
  • flagUrl, group (A–L)
  • fifaRanking
  • 48 équipes pré-seedées
MATCH
  • id, externalId (API football)
  • homeTeam → Team, awayTeam → Team
  • kickoffAt (UTC)
  • stage: GROUP | R32 | QF | SF | FINAL
  • group (A–L, nullable)
  • stadium, city
  • homeScore, awayScore (nullable)
  • status: SCHEDULED | LIVE | FINISHED
  • cutoffAt = kickoffAt − 5min
PREDICTION
  • id
  • user → User
  • match → Match
  • homeScore, awayScore
  • points (calculé après match)
  • updatedAt
  • UNIQUE(userId, matchId)
SCORING_RULE
  • id
  • exactScore: int (défaut: 3)
  • correctResult: int (défaut: 1)
  • wrongPrediction: int (défaut: 0)
  • bonusKnockout: int (défaut: 0)
  • updatedAt, updatedBy → User
LEADERBOARD_SNAPSHOT
  • userId → User
  • totalPoints
  • exactScores (count)
  • correctResults (count)
  • currentStreak
  • rank, computedAt
NOTIFICATION_LOG
  • id, userId → User
  • type: REMINDER | RESULT
  • channel: EMAIL | PUSH
  • matchId → Match
  • sentAt, status
